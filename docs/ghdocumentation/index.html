What was the idea behind making this site?

But firstly, how can you build it?

Don't be afraid. It's really simple once you get to know the basic things about building pages. The most important thing is to document everything you do. So that when someone else comes and would want to see what you have done and doesn't know the first thing about it, he can check out the README.md and have the essential components explained.

  • The basics

    • The same thing goes for building pages like this one. The knowledge you need to understand "how it's made" are basics of html, css, JQuery, JavaScript.

    All the above are explained in a very easy, clear when at Codecademy. You can take the courses and learn it, or move forward to the next step.

  • Github

    • The best place to store all of your work is Github. It is extremaly user-friendly. You simply sign-up, download a version of github suitable for your operating system --> MAC or Windows and you are good to go!

    What can you do here? Well... for starters you can "FORK" stuff form other people, which is really easy and it is also a very good way of getting to know the structure of the whole thing.

    To show you how easy it really is, chek out this 15 minutes starter at Github.

    After going through that we're pretty sure that you will have no problems with running your own github repository. But just to make sure you're fine, let us give you access to some cool pages that can make your gaining knowledge process easier

  • Jekyll

    • Adding Jekyll to your system is very useful, then you are able to build your website in a very easy and straight-forward way! You can add tabs, make default layouts for every tab without having to actually make a layout for every one separetaly!

    The main thing you need to do is to get to know your Terminal better. Become friends. I know it may sound funny, but it it neccesary in order to download all the components in a easy way.

    To download and use jekyll command in your terminal you need to download some other things. Firstly RVM or Homebrew

    {% highlight bash %} $ \curl -sSL https://get.rvm.io | bash -s stable or $ mkdir homebrew && curl -L https://github.com/Homebrew/homebrew/tarball/master | tar xz --strip 1 -C homebrew >in your /usr/local< {% endhighlight %}

    Nextly you will have to put in all these commands in your terminal, depending on what you are using staring with either 'rvm' or 'brew' --> like below.

    {%highlight bash %} $ brew install ruby $ brew install ruby-devel (ruby-dev on ubuntu) $ brew install rubygems $ brew install gcc $ gem install jekyll {% endhighlight bash %}

  • Oracle

    • What to do next? Well this is the moment when Oracle comes in handy. You simply go to the site and download 'instantclient: basic, sdk and sqlplus'. Choose the one compatible with your operating system.

    Now you need to create a directory in your terminal in /usr/opt/oracle/instantclient_version and unzip all of those instantclients there. While your current directory is /usr/opt/oracle/instantclient_version do the step below.

    After that you have to change the PATH, simply put commands below in your terminal

    {%highlight bash %} $ DYLD_LIBRARY_PATH=/opt/oracle/instantclient_version export DYLD_LIBRARY_PATH $ ORACLE_HOME=/opt/oracle/instantclient_version export ORACLE_HOME $ cd /opt/oracle/instantclient_version $ ln -s libclntsh.dylib.11.1 libclntsh.dylib $ gem install ruby-oci8 and also $ cp config/oracle_example.rb config/oracle.rb {%endhighlight bash %}

  • Back to github

    • Once you hav installed all the componets you can either create your own website (a litlle hard for the begining) or fork somebody's work and start to understand how the system works. We suggest to fork the site from jekyll to make sure how to build your own page based on all the necesary components.

    You simply have to sign-in and press FORK and then the whole directory is on your account. Now you have to go to it, and click on the Jekyll Repository and then click Clone in desktop! And now you have it on your desktop! When you make changes in the documents on your computer, they are automaticly synchronized with github, to change them permenantly on your account to simply have to Commit and it's done!

  • Jekyll once again

    • Now simply open your terminal --> go to the directory which contains your site, for example cd Desktop/Jekyll-master and then type:

    {%highlight bash %} $ ruby -rubygems db-test.rb $ jekyll build $ jekyll serve {%endhighlight bash %}

    This command has created a new directory in your project which is called _site. It is now ready to be thrown to the deep oceans of web.

    Now, just simply type localhost:4000 in your browser and the whole site is yours to explore!

  • MAMP

    • Alternatively you can use something esle, which we strongly recommend, which is MAMP. You can download it here. It is also a local server where you can see your website at work. Simply at the Apache Tab change the root to the newly made directory _site click Start Servers and your site is ready at localhost:8000
